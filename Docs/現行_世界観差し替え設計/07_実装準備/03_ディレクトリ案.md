# ディレクトリ案

最終更新: 2026-03-10

この文書は、新仕様を実装する際のディレクトリ分割案を整理する。
狙いは、旧プロトタイプと新仕様の責務を混ぜず、段階的に移行しやすくすることにある。

## 1. 方針

- 既存の `src/sim` と `src/storage` はすぐには壊さない
- 新仕様は別ディレクトリへ切って最小縦切りを先に通す
- 通ったあとで、必要に応じて旧実装を置き換える

## 3. 各ディレクトリの責務

### `src/domain/`

- 実装全体で共通に使う型
- ID 規則
- 列挙値
- JSON 正準形の定義

### `src/data/`

- 実在するサンプル世界観データ
- 最小実装で使う固定データ
- テスト用の初期セット

### `src/core/`

- 依頼進行の主処理
- 状態更新
- メモ候補生成
- 実装の中心ロジック

### `src/ai_contracts/`

- AI に渡す入力の形
- AI から受ける出力の形
- バリデーション補助

### `src/storage_v2/`

- 新仕様用の保存層
- 旧 `src/storage` とは切り離して扱う

### `src/examples/`

- 手動確認用のサンプル実行
- 開発中の通し確認

## 4. 旧実装との付き合い方

- `src/sim` は旧プロトタイプとして残す
- `src/storage` は `legacy` 扱いにするか、そのまま旧系として参照する
- 新仕様を `v2` 系ディレクトリへ切る
- 新系が通ってから必要なものだけ旧系から移す
