## 実行した確認

  - `mission_0001`

- 確認結果
  - 大成功を確認
  - `guildmaster_note` が 1 件保存されることを確認
  - 状態持ち越しは軽微で、次回依頼に進める

### 2. スタッフあり / 交渉寄り

- 対象依頼
  - `mission_0002`
- 期待したこと
  - `negotiator` と `scout` の意味が出るか
  - スタッフのコメントが人選補助として機能するか
- 確認結果
  - 成功を確認
  - 交渉役中心の編成が自然に選ばれる
  - 依頼後メモも 1 件以上残る

### 3. スタッフなし / 危険寄り

- 対象依頼
  - `mission_0003`

  - 失敗を確認
  - `resting` / `unavailable` への遷移を確認

### 4. 永続化

- 保存先
  - `/tmp/isdf_guild/cyberpunk_minimal_state.json`
- 確認結果
  - 1 サイクル後に保存
  - 再読込後に 2 サイクル目を継続実行できることを確認

## よかった点

- `mission` と `dispatch`、`report` と `guildmaster_note` の責務分離は保てている
- `snapshot` を `post_report` で切っており、次回への現在地として使いやすい
- 5 人の冒険者に役割差と性格差があり、依頼ごとの人選差が出る
- スタッフあり / なしの両方を最小コストで比較できる

## 現時点の違和感

- 成功系の `guildmaster_note` は今は簡易的で、まだ人物の深い主観差までは弱い
- `report.text` はテンプレート生成なので、世界観の乾いた文体は出ているが変化幅は狭い
- 関係更新は最小限で、人物同士の相性が強く効く段階にはまだ達していない
- `storage_v2` は JSON ファイル保存で、更新競合や履歴管理までは未対応

## 残る不明点

- `guildmaster_note` の `source_kind` は将来的に `rest_phase` や手動入力を含めるか
- `snapshot.faction_states[].relation_state` をどこまで列挙値で固定するか
- `facility` の効果を今後どこまでロジックへ反映するか
- `mission` の生成を固定データから AI 生成へ切り替える際、危険度と依頼数のバランスをどう保つか
- 永続化を今後も単一 JSON で続けるか、ストア分割を進めるか

## プレイ後に決めるべきもの

以下は、現時点では無理に先決めせず、実際に数サイクル回してから判断した方がよい論点である。

- スタッフごとの個性の強さ
  - どこまで観察力、口出し頻度、言い方の差を強めるか
- 火種の種類の追加範囲
  - 現在の最小セットで十分か、種類を増やした方が読み味がよくなるか
- 成功時余波の濃さ
  - 成功でもどれくらい後味や違和感を残すか
- 火種の残りやすさ
  - 人物性格による残留差が強すぎないか、弱すぎないか
- 関係変化の見え方
  - 会話やメモへの滲み方が読み取りやすいか、逆に説明不足すぎないか
- スタッフなし時の情報量
  - 冒険者会話だけで十分か、もう少し読み取り材料が要るか
- `report` の具体性
  - 事実中心のままいくか、場面描写や誰が何をしたかを増やすか
- `storage_v2` の強化優先度
  - 長期試運転前に保存層を強めるか、このまま挙動確認を優先するか

## プレイ中に注視すべき点

以下は、自由記述感想を書くときに意識して見ると判断材料になりやすい観点である。

### 1. 編成時の迷い

- 誰を出すかで本当に少し悩めるか
- 能力だけでなく、相性や消耗、因縁を自然に気にしたくなるか
- 「最適解を探す」より「なんとなく嫌な感じがする」が発生しているか

### 2. スタッフの台詞

- ただの説明文ではなく、そのスタッフらしい物言いに見えるか
- 直接言いすぎていないか
- 逆に曖昧すぎて何も拾えない感じになっていないか

### 3. スタッフなし時の読み味

- 説明が減ることで、逆に想像の余地が出ているか
- 冒険者会話や態度から何かを感じ取れるか
- 情報不足で単に分かりにくいだけになっていないか

### 4. 成功時の後味

- 成功でも「何かが少し残った」感じがあるか
- それが気持ちよさを壊しすぎていないか
- 逆に成功があっさり流れすぎていないか

### 5. `guildmaster_note`

- ただの結果要約ではなく、その人物をどう見たかになっているか
- 成功時メモでも人物差が少し感じられるか
- 同じ人物に毎回似たメモばかり出ていないか

### 6. 次回への余波

- 前回のことが、次の人選や気分に少しでも残っているか
- 何がどう影響したかを完全に説明されなくても、何となく筋が通っているか
- 読み取れた時に面白さがあるか

### 7. 読み取れなかった時の納得感

- うまく読めなかった結果として変なことが起きても、それが物語として許容できるか
- プレイヤーの誤読や見落としが、単なる理不尽ではなく「そういうこともある」に収まるか

### 8. 作業感の有無

- 依頼を1本片付けた時に、単なるルーチンに感じるかどうか
- 次の依頼に行く前に「少し考えたい」と思えるか
- その考えたくなる理由が、数字ではなく人物や空気に寄っているか
