---

このプロジェクトの実装を進めてください。

まず最初に `AGENTS.md` を読み、このプロジェクト固有ルールに従ってください。
次に、現行仕様の正本として以下を確認してください。

- `Docs/現行_世界観差し替え設計/00_案内.md`
- `Docs/現行_世界観差し替え設計/07_実装準備/01_実装全体プラン.md`
- `Docs/現行_世界観差し替え設計/07_実装準備/02_実装タスク表.md`
- `Docs/現行_世界観差し替え設計/07_実装準備/03_ディレクトリ案.md`
- `Docs/現行_世界観差し替え設計/07_実装準備/04_フェーズ1_型と正準データ構造の固定.md`

前提:

- 現在は実装前段階を終え、フェーズ1から着手できる状態です
- 最初に実装する世界観は `退廃サイバーパンク` です
- 最初に通すべき主ループは `mission -> dispatch -> report -> snapshot -> guildmaster_note` です
- 旧 `src/sim` と `src/storage` は旧プロトタイプなので、最初から直接改造しすぎないでください
- 新仕様は `src/domain/` `src/data/` `src/core/` `src/storage_v2/` などへ切る方針です

今回の実装対象:

- フェーズ1 「型と正準データ構造の固定」

今回やってほしいこと:

1. 現行仕様書と既存コードを確認し、フェーズ1の実装方針を短く整理する
2. `src/domain/` 配下に、新仕様用の ID / 列挙値 / 型定義の土台を作る
3. 少なくとも以下の型を定義する
   - `WorldPack`
   - `Faction`
   - `Character`
   - `StaffCharacter`
   - `Base`
   - `Facility`
   - `Mission`
   - `Dispatch`
   - `Report`
   - `Snapshot`
   - `GuildmasterNote`
4. 命名規則は `Docs/現行_世界観差し替え設計/05_補助資料/07_JSON命名規則とID規則.md` に従う
5. `mission` と `dispatch`、`report` と `guildmaster_note` を混ぜない
6. `obstacles` は配列で持つ正準形にする
7. 型の代表サンプルを最小限置き、`tsc` で型チェックする
8. 最後に、何を追加したか、次にフェーズ2で何をやるべきかを簡潔にまとめる

制約:

- まずはフェーズ1だけに集中してください
- UI 実装には進まないでください
- 永続化や AI 接続にはまだ進まないでください
- 旧 `src/storage/types.ts` の全面改修はしないでください
- 新仕様を無理に旧 `src/sim` に埋め込まないでください

実装時に重視してほしいこと:

- 型の正確さ
- 参照 ID の一貫性
- 後続フェーズが迷わない構造
- 仕様書とのズレを作らないこと

必要なら合理的な判断で進めて構いませんが、仕様とぶつかる箇所があれば明示してください。
作業完了後は、このプロジェクトのルールに従ってコミットしてください。
