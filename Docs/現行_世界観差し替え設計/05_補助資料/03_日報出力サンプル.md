# 日報出力サンプル

最終更新: 2026-03-10

この文書は、現行仕様に沿った日報出力の参考例を置く。
目的は以下。

- `text`、`intent_tags`、`reason_summary` の使い方を固める
- `guildmaster_note` をどの位置で挟むかを確認する
- 事実ログと主観メモを分離した出力例を持つ

## 1. 方針

日報は、まず「何が起こったか」を短く押さえる。
その上で、必要なら詳細本文を読む。

`guildmaster_note` は日報本文の一部ではなく、
依頼後にギルド主が残す別枠の人物メモとして扱う。

## 2. 最小出力例

```json
{
  "report_id": "report_0001",
  "mission_id": "mission_0007",
  "kind": "mission_report",
  "text": "旧港倉庫での封印箱護送は部分成功。箱は届けたが、帰路で護衛役の一人が負傷し、依頼人との信頼もわずかに揺らいだ。",
  "intent_tags": ["依頼結果", "部分成功", "負傷発生", "関係変化"],
  "reason_summary": "目的物の搬送自体は達成したが、消耗と不信が残った。"
}
```

## 4. `guildmaster_note` を含む流れの例

### 4.1 日報本文

```json
{
  "report_id": "report_0001",
  "mission_id": "mission_0007",
  "kind": "mission_report",
  "text": "旧港倉庫での封印箱護送は部分成功。箱は届いたが、帰路で護衛役のシオンが肩を負傷した。",
  "intent_tags": ["依頼結果", "部分成功", "負傷発生"],
  "reason_summary": "目的達成と引き換えに人的損耗が出た。"
}
```

### 4.2 AI が出す `guildmaster_note` 候補

```json
{
  "character_id": "char_shion",
  "note_candidates": [
    "任務はこなしたが、無理を押しているように見える。",
    "痛みを隠しているが、次も前に出たがるタイプだ。",
    "結果を優先しすぎる傾向があり、補佐役が必要かもしれない。"
  ]
}
```

### 4.3 プレイヤー選択後の保存例

```json
{
  "character_id": "char_shion",
  "guildmaster_note": {
    "selected_text": "痛みを隠しているが、次も前に出たがるタイプだ。",
    "user_note": "しばらく単独前衛は避けたい。",
    "created_from_report_id": "report_0001"
  }
}
```

## 5. 面談後メモの例

```json
{
  "character_id": "char_mina",
  "source_kind": "interview",
  "source_id": "interview_0004",
  "guildmaster_note": {
    "selected_text": "表向きは冷静だが、この件では珍しく感情を見せた。",
    "user_note": "港湾商会の話題に反応が強い。",
    "created_at_phase": "post_interview"
  }
}
```

## 6. 実装時の注意

- 日報本文は事実中心にする
- `intent_tags` は UI 表示より検索と再利用を優先する
- `reason_summary` は短く保つ
- `guildmaster_note` は評価確定ではなく、運用上の印象メモとして扱う
- `guildmaster_note` は上書きではなく履歴で積む方が相性がよい
