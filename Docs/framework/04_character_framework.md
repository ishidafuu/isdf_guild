# キャラクター能力設計 仕様草案

最終更新: 2026-03-11

## 1. 目的

本章は、世界観差し替え型ギルドゲームにおけるキャラクター設計の共通仕様を定義する。
狙いは以下の通り。

- 世界観が変わっても同じキャラ機能が通用する
- 能力値と役割を分離し、差し替えに強くする
- GURPS 的な汎用性は取り入れつつ、重量級になりすぎない
- 依頼解決とギルド運営の両方に効くデータ構造にする

## 2. 設計方針

- 能力値は少数の汎用軸に固定する
- 専門性は特技で出す
- キャラの弱さや癖は欠点として明示する
- 世界観差分は名称と演出に寄せる
- 数値計算よりも、依頼内での役割の違いが見えることを優先する
- プレイヤーはキャラ本人ではなく、ギルド側に近い立場から人材を運用する
- 能力値だけでなく、性格、欠点、関係性を踏まえて仕事を割り振る

## 3. キャラクターの共通構造

全キャラクターは以下の共通要素を持つ。

- 基本情報
- 能力値
- ロール
- 特技
- 欠点
- 世界観フレーバー要素
- 装備枠
- 状態値
- 信用や関係値
- 成長欄

## 4. 能力値の共通仕様

能力値は5種に固定する。
表示名は世界観ごとに差し替えてよいが、内部機能は変更しない。

### 4.1 `power`

- 主担当:
  - 戦闘、制圧、突破、重量物の取り扱い
- 主に使う依頼:
  - 討伐、護衛、防衛、救助

### 4.2 `tech`

- 主担当:
  - 工作、修理、解除、分析、機材操作
- 主に使う依頼:
  - 回収、輸送、調査、防衛

### 4.3 `sense`

- 主担当:
  - 索敵、追跡、観察、探索、異常察知
- 主に使う依頼:
  - 調査、救助、討伐、回収

### 4.4 `social`

- 主担当:
  - 交渉、説得、聞き込み、場の調整、印象形成
- 主に使う依頼:
  - 交渉、護衛、調査、輸送

### 4.5 `will`

- 主担当:
  - 精神抵抗、継戦、集中、恐怖耐性、信念の維持
- 主に使う依頼:
  - 全カテゴリ

## 5. 能力値の推奨レンジ

軽量運用を優先し、初期値は以下のいずれかで扱う。

### 5.1 シンプル型

- 0 から 3 の4段階
- 0: 苦手
- 1: 平均
- 2: 得意
- 3: 専門

### 5.2 しっかり型

- 1 から 5 の5段階
- 1: 明確に弱い
- 2: やや弱い
- 3: 標準
- 4: 強み
- 5: 一線級

現段階では、世界観差し替えとの相性と扱いやすさから `0-3` のシンプル型を推奨する。
知識系は独立能力値として分けず、当面は `tech` と `sense` と `social` に分散して扱う。

## 6. ロール機能

ロールは能力値とは別に定義する。
ロールの役目は「そのキャラが任務中に何を担当するか」を見せることにある。
ロールは固定職ではなく、ゆるい得意分野ラベルとして扱う。

- `frontliner`
  - 前線役。脅威の受け止め、排除、突破を担う
- `support`
  - 支援役。回復、強化、保護、継戦補助を担う
- `scout`
  - 探索役。発見、索敵、追跡、先行確認を担う
- `engineer`
  - 技術役。解除、修理、分析、特殊装置処理を担う
- `negotiator`
  - 渉外役。交渉、聞き込み、説得、折衝を担う

表示名は世界観ごとに変更してよい。

補足:
1キャラは主ロールを1つ持つが、他ロール相当の行動を完全には禁止しない。

## 7. 特技の仕様

特技は「能力値では表現しきれない専門性」を示す。
特技はルール処理上、以下のいずれかの働きを持つ。

- 特定場面で判定補正を与える
- 特定カテゴリの障害を無視または軽減する
- 追加情報や追加選択肢を開く
- 支援や連携を可能にする
- 失敗時の損害を軽減する

### 7.1 特技の分類

- 共通特技
  - 世界観に依存しない機能特技
- 世界観フレーバー特技
  - 同じ機能だが名称と演出だけが異なる特技
- 世界観固有特技
  - その世界でのみ成立する特殊要素

### 7.2 共通特技テンプレ

- 戦術打撃
  - 討伐・防衛で単体の強敵に強い
- 制圧姿勢
  - 護衛・防衛で敵の突破を止めやすい
- 応急処置
  - 負傷や損耗を軽減する
- 現場修理
  - 壊れた装置や道具をその場で復旧する
- 痕跡読解
  - 調査・追跡時に追加情報を得る
- 危険察知
  - 奇襲や罠への初動に強い
- 交渉術
  - 交渉・聞き込み時の成功率を上げる
- 顔の広さ
  - 人脈経由の情報取得や依頼導線に強い
- 搬送技術
  - 輸送・救助で損耗を減らす
- 封印処理
  - 回収や防衛で危険物対応ができる
- 指揮補佐
  - 仲間の判定補助やチーム安定化に寄与する
- 不屈
  - `will` を使う判定で粘りが利く

### 7.3 特技記述テンプレ

各特技は以下の形式で記述する。

```md
- 特技名:
- 分類:
- 機能:
- 有効カテゴリ:
- 条件:
- 演出例:
```

## 8. 欠点の仕様

欠点は、キャラの弱さや癖が依頼解決にどう影響するかを示す。
欠点は「ただ不利なだけ」ではなく、キャラらしさと選択の重みを生むために使う。
演出専用ではなく、判定や選択肢にも明確に影響させる。

### 8.1 欠点の機能

- 特定条件下で不利補正を受ける
- 特定判断をしづらくなる
- 特定依頼との相性が悪くなる
- 物語上の火種や合併症を生む

### 8.2 欠点テンプレ

- 短気
  - 挑発や侮辱に弱く、交渉がこじれやすい
- 過剰慎重
  - 期限の厳しい依頼で動きが遅れる
- 名誉優先
  - 裏仕事や秘匿任務と相性が悪い
- 臆病
  - 高圧的な脅威や恐怖演出に弱い
- 疑い深い
  - 仲間や依頼人との関係構築が遅い
- お人好し
  - 嘘や罠に巻き込まれやすい
- 衝動的
  - 調査や交渉で先走りやすい
- 過去の因縁
  - 特定勢力やカテゴリで平常心を失う
- 身体不安
  - 長時間行動や負傷に弱い
- 執着
  - 特定対象を前に優先順位を見失いやすい
- 潔癖
  - 怪異、汚染、裏社会など特定演出に弱い
- 孤立傾向
  - 支援や連携を活かしにくい

### 8.3 欠点記述テンプレ

```md
- 欠点名:
- 種別:
- 発動条件:
- 不利益:
- 演出例:
```

## 9. 性格タグと関係値

能力値や特技だけではキャラ差が薄くなるため、軽量な性格タグを推奨する。

### 9.1 性格タグ

2個から3個程度を持たせる。

- 冷静
- 粗野
- 面倒見がよい
- 功利的
- 信仰心が厚い
- 負けず嫌い
- 観察好き
- 皮肉屋
- 純朴
- 野心家

補足:
性格タグや欠点は、プレイヤーが自由選択するものではなく、キャラ生成時にランダムまたは半ランダムで付与される前提とする。
プレイヤーは生成された性格や欠点を見て運用する。

### 9.2 関係値

少なくとも以下を保持できる形が望ましい。

- ギルドとの信頼
- 主要NPCとの関係
- 同行メンバーとの相性
- 特定勢力への感情

補足:
関係値は補助的な味付けではなく、各キャラに持たせる主要要素として扱う。
依頼紹介、同行相性、受諾判断、余波の演出に継続的に使う。

## 10. 成長の仕様

成長は以下の3層で扱う。

- 能力値成長
  - 長期的で緩やか
- 特技習得
  - キャラ差が出やすい主要成長
- 関係値・評判成長
  - ギルド運営や依頼導線に影響する

推奨としては、能力値上昇は低頻度に抑え、特技取得と人間関係変化を主な成長実感にする。

### 10.1 成長の主役

本作における成長の主役は、キャラクター同士の物語と関係変化である。

- 誰と誰が信頼を深めたか
- 誰と誰が衝突したか
- どの依頼で印象が変わったか
- どの失敗や成功が人物像を更新したか

これらを、能力値上昇よりも強い継続動機として扱う。

### 10.2 ギルド成長の位置づけ

ギルド拡張は重要な軸ではあるが、常に主目的とはしない。

- 拡張したければ施設、人員、信用、依頼幅を伸ばせる
- 拡張を急がず、人物運用や関係性の変化を主に楽しんでもよい
- 経営要素は、物語の舞台を広げる任意の手段として置く

要するに、ギルド経営は必須の勝利条件ではなく、物語の器を広げる可変要素として扱う。

### 10.3 成長要素の見せ方

成長要素には以下がある。

- 能力値上昇
- 特技取得
- 装備強化
- 信頼関係の変化
- 地位上昇

本作では、これらに強い優先順位を機械的に付けすぎない。
プレイヤーは冒険者本人ではなく、ギルド側から人物や案件を見ているため、どれか1つだけを追いかける構造よりも、自然に積み重なる構造を優先する。

推奨される目立ち方の順序は以下の通り。

- 信頼関係の変化
  - 最も目立つ。物語の主役として扱う
- 特技取得
  - 個性の伸びとしてわかりやすく扱う
- 地位上昇
  - ギルド内外での立ち位置変化として自然に付いてくる
- 装備強化
  - 必要に応じて行うが、主目的にはしない
- 能力値上昇
  - 低頻度で、補助的に起こるものとする

補足:
これは厳密な報酬優先順位ではなく、「プレイ中に自然に目立つ順番」の目安である。
数値の強化よりも、人間関係と立場の変化が前に来る構造を推奨する。

### 10.4 恒久損失の扱い

死亡や引退のような恒久損失は、ゲーム全体の前提として固定しない。
キャンペーン開始時に選ぶ設定として扱う。

推奨パターン:

- 恒久離脱なし
  - 負傷や関係悪化は残るが、死亡や強制退場は起きない
- 引退あり
  - 蓄積した損耗や事情によって、戦線離脱や離籍が起こりうる
- 死亡あり
  - 任務失敗や重大事故により、死亡や不可逆の喪失が起こりうる

補足:
どの設定でも、恒久損失は単なる罰ではなく、物語の温度感を決める要素として扱う。
このゲームではクリア条件より継続運用が重要なため、開始時にトーンを選べる設計が望ましい。

## 10.5 初期キャラクターの前提

初期キャラクターは、一律に有能なプロにも、一律に半人前にも固定しない。
ギルド所属人材として、力量が混在していてよい。

- 新人
- 実務に慣れた中堅
- 癖の強い経験者
- 扱いづらいが特定分野で刺さる人材

推奨方針:

- 序盤から依頼遂行は可能
- ただし高難度依頼を安定周回できるほど完成されてはいない
- 強みと弱みの混在が、配置判断の面白さを生む

## 11. 世界観差し替え対応

世界観パックは以下を差し替えてよい。

- 能力値の表示名
- ロールの表示名
- 特技名
- 欠点名
- 状態異常名
- 装備名
- キャラ出自名

ただし以下は変更しない。

- 能力値の内部機能
- ロールの担当機能
- 特技の処理類型
- 欠点の不利益構造

## 14. 初期作成ルールのたたき台

- ロールを1つ決める
- 能力値5種に合計6点から8点を割り振る
- 特技を3つ決める
- 欠点を1つから2つ生成する
- 性格タグを2つから3つ生成する
- 関係値の初期状態を決める
- 世界観フレーバー要素を1つ決める

補足:
プレイヤーが手作業でビルドするよりも、生成された人材を受け取って運用する形を基本とする。

## 15. 設計メモ

- GURPS 的な参考点は「抽象能力値」「長所短所」「世界観横断性」であり、技能細分化ではない
- 依頼生成との噛み合わせを優先し、技能一覧の肥大化は避ける
- 欠点は物語上のフックとしても使えるようにしておく
