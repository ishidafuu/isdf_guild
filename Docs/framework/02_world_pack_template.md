## 3. 生成手順の簡易仕様

1. `時代感` を振る
2. `ギルドの立場` を振る
3. `主脅威` を振る
4. `報酬価値` を振る
5. `拠点種別` を振る
6. `雰囲気タグ` を振る
7. 結果をつなぎ、依頼カテゴリ8種に自然に言い換えられるか確認する
8. 不自然なら1項目だけ振り直す
9. 最後に `固有ギミックを1つだけ` 人力で足す

この最後の1つだけ手作業で足す工程で、完全ランダムではない作者性を残す。

## 4. 初期ワールド運用方針

初期ワールドの本数は、あらかじめ厳密に固定しない。
まずは代表的なパックをいくつか作って試運転し、追加の世界観も同じ枠組みで自然に増やせるかを確認する。

推奨順:

1. 王道ファンタジーで基準挙動を確認する
2. サイバーパンクで語彙差し替えとプレイ感の変化を確認する
3. 和風怪異でトーン差と物語運用を確認する
4. 問題がなければ、他世界観を段階的に追加する

## 5. 固有要素の許容方針

世界観固有特技や固有資源は、その世界観に合っている限り広めに許容する。

例:

- ファンタジーの祝福、呪具、魔石
- サイバーパンクの改造、アクセス権、神経負荷
- 和風怪異の結界、守札、穢れ

ただし、以下は守る。

- 共通能力値や共通依頼カテゴリを置き換えない
- 判定方式そのものを変えない
- 他世界観へ無理に移植しなくてよい

要するに、固有要素は「その世界だけの濃い味」として積極的に持たせてよい。

## 6. 物語トーンの扱い

物語トーンは、作者が固定してもよいし、ユーザーが開始時に選んでもよいし、ランダムに決めてもよい。
以下のような軸を独立に持たせると扱いやすい。

- 明るさ
- 苦味
- コメディ許容度
- 残酷さ
- 理不尽さ
- 中心感情

これらは世界観そのものに結びつけてもよいが、同じ世界観でも別のトーンで遊べる余地を残す。
