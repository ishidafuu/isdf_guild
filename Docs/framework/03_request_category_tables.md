# 依頼カテゴリ8種の共通仕様と生成表

最終更新: 2026-03-10

## 1. 目的

依頼を世界観依存の文章ではなく、機能単位で管理する。
各世界観では、この機能単位の依頼に対して名前・対象・演出を上書きする。

内部的には同じ依頼でも、表示上は以下のように差し替えられる。

- ファンタジー: 魔物討伐
- サイバーパンク: 排除任務
- 和風怪異: 怪異鎮め

## 2. 共通依頼カテゴリ

- `subjugation` 討伐
- `escort` 護衛
- `investigation` 調査
- `recovery` 回収
- `delivery` 輸送
- `negotiation` 交渉
- `defense` 防衛
- `rescue` 救助

## 3. 依頼の共通構造

すべての依頼は以下の項目で定義する。

- `category`
- `client`
- `target`
- `location`
- `objective`
- `obstacle`
- `time_limit`
- `reward`
- `risk`
- `twist`
- `difficulty`
- `tags`

## 4. カテゴリ別機能定義

### 4.1 討伐 `subjugation`

- 目的:
  - 脅威対象を排除、無力化、鎮静化する
- 成功条件:
  - 対象が活動不能になる
- 主な障害:
  - 戦闘、追跡、接近、対象特性

### 4.2 護衛 `escort`

- 目的:
  - 人物、物資、施設、行列などを安全に守る
- 成功条件:
  - 対象が損害なく目的地または期限まで維持される
- 主な障害:
  - 奇襲、妨害、裏切り、地形

### 4.3 調査 `investigation`

- 目的:
  - 情報、原因、所在、真相を明らかにする
- 成功条件:
  - 必要情報の取得
- 主な障害:
  - 隠蔽、偽情報、危険地帯、証拠不足

### 4.4 回収 `recovery`

- 目的:
  - 特定の物品、情報、人材、遺物を確保する
- 成功条件:
  - 対象を回収し持ち帰る
- 主な障害:
  - 所有者、封印、警備、損傷リスク

### 4.5 輸送 `delivery`

- 目的:
  - 対象物を指定地点へ運ぶ
- 成功条件:
  - 期限内に対象を搬送する
- 主な障害:
  - 襲撃、天候、故障、検問、地形

### 4.6 交渉 `negotiation`

- 目的:
  - 対立する相手との合意、取引、停戦、説得を成立させる
- 成功条件:
  - 条件を満たした合意形成
- 主な障害:
  - 利害衝突、感情対立、第三者妨害、情報不足

### 4.7 防衛 `defense`

- 目的:
  - 一定時間または一定条件まで対象地点・人物・設備を守る
- 成功条件:
  - 防衛対象の保持
- 主な障害:
  - 波状攻撃、資源不足、内部崩壊、地形不利

### 4.8 救助 `rescue`

- 目的:
  - 危機にある対象を発見し保護する
- 成功条件:
  - 対象の生存確保または離脱完了
- 主な障害:
  - 行方不明、時間制限、精神不安定、追手

## 5. 依頼生成テンプレ

`依頼人` が、`場所` で発生している `対象/問題` に対して、`目的` の達成を求める。
主な障害は `障害` であり、期限は `期限`。
成功時には `報酬`、失敗時には `不利益` が発生する。
途中で `追加事情` が判明する可能性がある。

## 6. 依頼カテゴリ別生成表

### 6.1 討伐 `subjugation`

基本構文:
`依頼人` が、`場所` に現れる `脅威対象` の排除を依頼する。被害は `被害内容`。障害は `厄介要素`。報酬は `報酬`。

必須項目:
- 依頼人
- 場所
- 脅威対象
- 被害内容
- 厄介要素
- 報酬

生成表:

`依頼人`
- 村人・住民
- 商人・運送業者
- 地方役人・管理者
- 宗教関係者
- 研究者・学者
- 同業組織
- 被害者本人
- 正体を隠した依頼人

`場所`
- 街道沿い
- 廃墟
- 森林地帯
- 地下区画
- 水辺・港湾
- 山岳地帯
- 遺跡・旧施設
- 居住区の近辺

`脅威対象`
- 人を襲う単独強敵
- 群れを成す小型脅威
- 指揮個体・親玉
- 暴走した守護装置
- 呪いや怪異の核
- 逃げ回る高機動対象
- 潜伏型の暗殺者
- 定期的に現れる災害存在

`被害内容`
- 人命被害が出ている
- 物流が止まっている
- 立ち入りが不可能になっている
- 住民が避難している
- 施設が破壊されている
- 調査隊が戻ってこない
- 特産品や資源が採れない
- 重要行事が中止寸前

`厄介要素`
- 出現条件が不明
- 夜間しか活動しない
- 周囲に二次被害を出す
- 通常手段が効きにくい
- 地形が悪い
- 囮や分身を使う
- 他勢力が介入してくる
- 排除より捕獲が望ましい

`報酬`
- 金銭
- 希少資源
- 信用・名声
- 特別許可
- 情報
- 装備支援
- 人脈
- 拠点機能の解放

`ひねり候補`
- 本当の脅威は別にいる
- 対象は人為的に放たれていた
- 依頼人が事実を隠している
- 討伐対象は本来無害だった
- 排除すると別の問題が発生する
- 競合組織も同じ依頼を受けている

### 6.2 護衛 `escort`

基本構文:
`依頼人` が、`護衛対象` を `目的地または期間` まで守るよう依頼する。脅威は `想定妨害`。制約は `護衛条件`。報酬は `報酬`。

必須項目:
- 依頼人
- 護衛対象
- 目的地または期間
- 想定妨害
- 護衛条件
- 報酬

生成表:

`護衛対象`
- 要人
- 商隊・輸送便
- 儀式や式典の参加者
- 研究資料や試作品
- 怪我人・避難民
- 証人
- 交渉使節
- 封印物・危険物

`目的地または期間`
- 隣町まで
- 危険地帯の通過完了まで
- 一晩の間
- 式典終了まで
- 契約締結まで
- 調査終了まで
- 船便到着まで
- 追手を撒くまで

`想定妨害`
- 盗賊・敵対勢力の襲撃
- 内通者による妨害
- 自然環境の悪化
- 対象自身の問題行動
- 群衆混乱
- 監視や検問
- 待ち伏せ
- 誤情報による誘導

`護衛条件`
- 対象に正体を明かせない
- 対象を傷つけられない
- 荷物を開封できない
- 派手な戦闘は禁止
- 期限厳守
- 護衛人数に制限がある
- 別件を並行処理する必要がある
- 敵対勢力に見つかってはならない

`報酬`
- 金銭
- 安全通行権
- 追加依頼の優先権
- 評判上昇
- 保護契約
- 珍しい品
- 後援者の獲得
- 勢力との関係改善

`ひねり候補`
- 護衛対象が重要情報を隠している
- 本命は対象ではなく荷物
- 守る相手が依頼側を信用していない
- 目的地自体が罠
- 護衛対象が実は囮
- 敵より対象自身の行動が危険

### 6.3 調査 `investigation`

基本構文:
`依頼人` が、`異変や事件` について `必要情報` を明らかにするよう依頼する。手掛かりは `初期情報`、障害は `妨害要因`、報酬は `報酬`。

必須項目:
- 依頼人
- 異変や事件
- 必要情報
- 初期情報
- 妨害要因
- 報酬

生成表:

`異変や事件`
- 失踪事件
- 連続被害
- 原因不明の事故
- 不審な噂の拡大
- 設備異常
- 資源枯渇
- 領域汚染・呪染
- 内部不正の疑い

`必要情報`
- 原因の特定
- 犯人や元凶の特定
- 発生地点の特定
- 次の被害予測
- 関係者の洗い出し
- 隠蔽の証拠
- 解決法の発見
- 真偽の確認

`初期情報`
- 目撃証言がある
- 断片的な記録が残っている
- 唯一の生存者がいる
- 現場に不自然な痕跡がある
- 古い類似事件がある
- 依頼人が一部を知っている
- 競合勢力も動いている
- 情報そのものが禁制扱い

`妨害要因`
- 証言が食い違う
- 証拠が偽装されている
- 関係者が口を閉ざす
- 現地が危険地帯
- 時間経過で痕跡が消える
- 公権力や上位勢力が介入する
- 原因が複数重なっている
- 調査行為自体が敵を刺激する

`報酬`
- 調査費
- 情報アクセス権
- 依頼人の信頼
- 追加案件の優先紹介
- 封印情報の閲覧権
- 特殊装備の貸与
- 勢力内の発言権
- 重要人物への紹介

`ひねり候補`
- 異変は事故ではなく誘導されている
- 元凶は依頼人側にいる
- 真相を公表すると別の損害が出る
- 調査対象は脅威ではなく被害者
- 一番怪しい存在は無関係
- 解決には戦闘以外の処理が必要

### 6.4 回収 `recovery`

基本構文:
`依頼人` が、`回収対象` を `所在` から確保するよう依頼する。問題は `保護要因または所有者`。条件は `回収制限`。報酬は `報酬`。

必須項目:
- 回収対象
- 所在
- 保護要因または所有者
- 回収制限
- 依頼人
- 報酬

生成表:

`回収対象`
- 古い遺物
- 機密データ
- 試作品
- 呪物
- 医療物資
- 契約書や証拠文書
- 行方不明の専門家
- 希少素材

`所在`
- 敵勢力の拠点
- 危険地帯の奥
- 廃棄施設
- 市場・オークション
- 封鎖区域
- 個人の邸宅や研究室
- 移動中の輸送便
- 異常空間や結界内

`保護要因または所有者`
- 重警備
- 封印
- トラップ
- 法的保護
- 敵対組織の管理
- 所有者本人が手放さない
- 対象が壊れやすい
- 回収時に周囲へ害を及ぼす

`回収制限`
- 無傷で持ち帰る
- 秘密裏に行う
- 証拠を残さない
- 使用者ごと保護する
- 期限内必須
- 第三者に渡してはならない
- 解析せず回収する
- 破壊せず封印状態で運ぶ

`報酬`
- 高額報酬
- 特別装備
- 取引先の紹介
- 依頼人組織との関係強化
- 施設利用権
- 限定情報
- 次回大型案件の参加資格
- 希少資源

`ひねり候補`
- 回収対象が意思を持っている
- 本物と偽物がある
- 所有者にも正当な言い分がある
- 回収対象が運び出すと起動する
- 依頼人は回収後の用途を隠している
- 対象よりも同行者が重要

### 6.5 輸送 `delivery`

基本構文:
`依頼人` が、`輸送物` を `出発地` から `到着地` へ `期限` までに届けるよう依頼する。障害は `輸送上の問題`。報酬は `報酬`。

必須項目:
- 依頼人
- 輸送物
- 出発地
- 到着地
- 期限
- 輸送上の問題
- 報酬

生成表:

`輸送物`
- 食料・生活物資
- 医療品
- 危険物
- 契約品
- 重要部品
- 供物・祭具
- 身分秘匿中の人物
- 情報媒体

`出発地`
- 拠点都市
- 辺境集落
- 港
- 旧施設
- 隠し倉庫
- 研究区画
- 寺社・儀式場
- 他勢力の中立地帯

`到着地`
- 前線拠点
- 孤立集落
- 避難所
- 取引場所
- 研究機関
- 要塞・検問の外
- 儀式現場
- 秘密の受け渡し地点

`期限`
- 夜明けまで
- 3日以内
- 次の襲撃前まで
- 儀式開始前まで
- 追手到着前まで
- 天候悪化前まで
- 燃料・鮮度が尽きる前まで
- 次便出発前まで

`輸送上の問題`
- 襲撃予告がある
- 天候や環境が悪い
- 検問を越える必要がある
- 輸送物が不安定
- 道中に裏切り者がいる疑い
- 移動手段が限られる
- ルート上に脅威が巣食う
- 到着先が本当に安全かわからない

`報酬`
- 金銭
- 物資分配権
- 航路・通行許可
- 勢力評価
- 継続契約
- 地域住民からの信頼
- 追加支援
- 特権的な取引条件

`ひねり候補`
- 輸送物の中身が聞いていたものと違う
- 到着地がすでに陥落している
- 本当の目的は囮輸送
- 依頼人が受取人を信用していない
- 輸送物自体が周囲に影響を与える
- 一番安全なルートが一番遅い

### 6.6 交渉 `negotiation`

基本構文:
`依頼人` が、`交渉相手` との間で `争点` を解決するよう依頼する。必要なのは `妥結条件`。障害は `対立要因`。報酬は `報酬`。

必須項目:
- 依頼人
- 交渉相手
- 争点
- 妥結条件
- 対立要因
- 報酬

生成表:

`交渉相手`
- 地方権力者
- 商会・企業代表
- 宗教関係者
- 地域共同体の長
- 敵対勢力の使者
- 中立組織
- 怪異や異種族の代表
- 内部反対派

`争点`
- 通行権
- 資源分配
- 停戦
- 捕虜や人質の返還
- 汚染地帯や禁足地の扱い
- 責任の所在
- 納品条件
- 組織の活動許可

`妥結条件`
- 双方の損失を抑える
- 一時合意だけでもよい
- 書面化が必要
- 第三者証人が必要
- 代替案の提示が必要
- 依頼人側の譲歩が必要
- 相手の面子を保つ必要がある
- 裏条件を飲ませる必要がある

`対立要因`
- 感情的対立
- 過去の因縁
- 嘘をついている側がいる
- 相手が複数派閥に割れている
- 外部勢力が妨害する
- 交渉時間が短い
- 条件が現実的でない
- 失敗すると即衝突になる

`報酬`
- 信頼
- 活動許可
- 資源配分
- 勢力友好
- 情報開示
- 後援
- 安全保障
- 政治的立場の確保

`ひねり候補`
- 交渉相手は本音では合意したい
- 実際の決定権者は別人
- 争点そのものが誤認
- 条件を飲むと長期的損失が出る
- 戦うより不利な取引を迫られる
- 第三者が交渉決裂を望んでいる

### 6.7 防衛 `defense`

基本構文:
`依頼人` が、`防衛対象` を `時間または条件` の間守るよう依頼する。脅威は `攻撃側`。難所は `防衛上の不利`。報酬は `報酬`。

必須項目:
- 依頼人
- 防衛対象
- 時間または条件
- 攻撃側
- 防衛上の不利
- 報酬

生成表:

`防衛対象`
- 集落
- 拠点施設
- 倉庫
- 儀式場
- 重要人物
- 検問所
- 通信設備
- 結界・封印装置

`時間または条件`
- 夜明けまで
- 援軍到着まで
- 儀式完了まで
- 住民避難完了まで
- 修理完了まで
- 次の便出発まで
- 結界安定化まで
- 指揮官の帰還まで

`攻撃側`
- 群れ型の脅威
- 組織的な敵部隊
- 内部攪乱を狙う工作員
- 遠距離攻撃主体の敵
- 正面突破型の強敵
- 波状攻撃
- 怪異的・不可視の脅威
- 災害と敵襲の複合

`防衛上の不利`
- 人手不足
- 補給不足
- 地形不利
- 防衛対象が脆い
- 中に裏切り者がいる疑い
- 同時多発地点がある
- 民間人を抱えている
- 設備が老朽化している

`報酬`
- 防衛費
- 住民の支持
- 施設利用権
- 追加戦力の提供
- 物資供給の優先権
- 指揮権の拡大
- 拠点強化支援
- 長期契約

`ひねり候補`
- 真の目的は時間稼ぎではない
- 敵の狙いは破壊でなく奪取
- 守るべき対象が複数ある
- 外からの敵より内部崩壊が危険
- 防衛成功後に追撃判断を迫られる
- 防衛対象が守る価値のあるものか揺らぐ

### 6.8 救助 `rescue`

基本構文:
`依頼人` が、`救助対象` を `危険状況` から助け出すよう依頼する。場所は `救助地点`、問題は `救助困難要因`、報酬は `報酬`。

必須項目:
- 依頼人
- 救助対象
- 危険状況
- 救助地点
- 救助困難要因
- 報酬

生成表:

`救助対象`
- 行方不明者
- 取り残された住民
- 人質
- 負傷した重要人物
- 調査隊
- 子どもや弱者
- 捕縛された仲間
- 精神的に不安定な生還者

`危険状況`
- 脅威に包囲されている
- 閉じ込められている
- 時間経過で悪化する
- 汚染や呪詛に侵されている
- 追手が迫っている
- 輸送中に奪われた
- 災害現場に取り残された
- 敵地深くにいる

`救助地点`
- 崩落現場
- 敵拠点
- 森や山中
- 水辺・海上
- 廃施設
- 地下区画
- 結界や封印領域
- 混乱中の市街地

`救助困難要因`
- 正確な位置が不明
- 救助対象が動けない
- 救助対象が協力的でない
- 二次被害が大きい
- 時間制限が厳しい
- 敵が見張っている
- 搬送手段が不足
- 救助後の離脱が難しい

`報酬`
- 謝礼
- 信頼
- 借り
- 重要証言
- 勢力との関係改善
- 非公開情報
- 希少物資
- 仲間や人材の加入

`ひねり候補`
- 救助対象が秘密を抱えている
- 助ける相手が複数いて全員は無理
- 対象は救助を拒む理由がある
- 救助対象が敵側に利用されている
- 本当の救助対象は別にいる
- 搬送中に状況が反転する

## 7. 全カテゴリ共通の追加事情表

- 依頼人が一部の情報を伏せている
- 競合する別依頼と衝突する
- 対象は単純な悪ではない
- 第三勢力が横取りを狙う
- 時間制限が途中で短縮される
- 成功条件と倫理が衝突する
- 依頼達成が別地域に悪影響を与える
- 味方の中に利害不一致がある
- 現場で想定外の災害が起きる
- 依頼そのものが試し行為である
