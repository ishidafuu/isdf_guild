# 次に確認したい質問

最終更新: 2026-03-10

## 1. 目的

本章は、`framework` 側の仕様を次段階へ進めるために、ユーザー回答を前提として詰めたい論点を整理したものである。
可能な限り、答えると仕様が前進する質問に寄せている。

## 2. コアルール関連

- 判定方式は最終的に `2D6` 系で行きたいですか。それとも `%` や d20 もまだ候補ですか。
- 成功段階は `大成功 / 成功 / 失敗` の3段階で十分ですか。部分成功を独立させたいですか。
- 戦闘は「依頼中の1障害」として軽く扱いたいですか。それとも独立した見せ場にしたいですか。
- セッション1回の長さは、短編依頼1本を想定していますか。それとも数件をまとめて回す想定ですか。

## 3. キャラクター設計関連

- 能力値5種で足りますか。知識系を独立させたい気持ちはありますか。
- ロールは固定役職に近いですか。それともゆるい得意分野ラベルにしたいですか。
- 初期キャラはある程度有能なプロですか。それとも半人前スタートですか。
- 欠点はどの程度強く効かせたいですか。演出中心ですか、判定にも明確に効かせたいですか。
- 性格タグや欠点はプレイヤーが選ぶ形式ですか。それとも半ランダムにしたいですか。
- キャラの関係性は主軸にしたいですか。それとも補助的な味付けに留めたいですか。

## 4. 成長関連

- 成長の主役は何にしたいですか。
- 能力値上昇、特技取得、装備強化、信頼関係、地位上昇のうち優先したい順番はどうですか。
- 死亡や引退のような恒久損失は入れたいですか。
- 負傷やストレスはセッションをまたいで残したいですか。
- ギルド成長と個人成長はどちらを強く感じさせたいですか。

## 5. 世界観差し替え関連

- 初期ワールドは最終的に何本用意したいですか。
- 王道ファンタジーは基準世界として残しますか。それとも並列の1候補として扱いますか。
- 世界観差し替えは「完全に別世界」ですか。それとも同じ多元世界の別地域のようにつなげたいですか。
- 世界観ごとの差は、見た目と語彙の差で十分ですか。ある程度プレイ感まで変えたいですか。
- 世界観固有特技や固有資源は、どのくらい許容したいですか。

## 6. 依頼生成関連

- 依頼カテゴリ8種で不足感はありますか。採取や潜入を独立カテゴリにしたいですか。
- 依頼生成は完全ランダム寄りですか。GM 的な手直し前提ですか。
- 依頼には毎回「隠し事」や「追加事情」を入れたいですか。それとも軽い日常依頼も混ぜたいですか。
- 失敗時の余波はどの程度まで重くしたいですか。次回に軽く残る程度か、長く尾を引く形か。
- 依頼人や勢力との関係で、案件内容が変質する構造を強めたいですか。

## 7. ギルド運営関連

- ギルド運営はどこまで細かくしたいですか。施設、財政、人員、評判、派閥関係のどこが主軸ですか。
- 拠点拡張は数値的な成長ですか。それとも施設カードや設備一覧のように見せたいですか。
- ギルドが大きくなると何が増えるのが理想ですか。依頼数、上位案件、政治性、抱えるリスクのどれですか。
- プレイヤーの気持ちとして、街や組織を育てる感覚をどの程度重視したいですか。

## 8. 物語トーン関連

- 全体として明るい冒険寄りですか。少し苦い群像劇寄りですか。
- コメディや軽さはどの程度まで許容しますか。
- 残酷さや理不尽さはどのくらいまで入れたいですか。
- プレイヤーに感じてほしい中心感情は何ですか。
- 達成感、哀愁、緊張、連帯感、成り上がり感、世界の謎への接近のどれが近いですか。

## 9. 実装・運用関連

- 将来的にこの辞書や生成表は、実装で JSON やデータテーブルに落とす前提ですか。
  - 確定: データテーブル化前提
- AI 生成はどこまで任せたいですか。世界観、依頼文、NPC、日報、キャラ背景のうち優先度はどうですか。
  - 確定: ゲーム開始後は基本的に全般を AI 主導
- 人間が固定したい部分と、AI に委ねたい部分の境界はありますか。
  - 確定: 世界観の入口条件や遊びたくない方向の除外は人間が決める
- ソロプレイ寄りですか。複数人TRPG的運用も視野に入れていますか。
  - 確定: ソロプレイ前提

## 10. 個人的に特に答えがほしい質問

- あなたにとってこのゲームの主役は「依頼」なのか、「キャラ」なのか、「ギルド」なのか。
- 世界観を差し替えられること自体を商品価値として前面に出したいですか。
- このゲームで一番避けたい失敗は何ですか。
- プレイ後に「このゲームらしかった」と感じる要素を一言で言うと何ですか。
- ファンタジー世界観を基準にしつつ広げるのか、最初から完全に多世界前提で設計するのか、今の気持ちはどちらに近いですか。
