# 次に確認したい質問

最終更新: 2026-03-11

## 1. 目的

本章は、`framework` 側の仕様を次段階へ進めるために、ユーザー回答を前提として詰めたい論点を整理したものである。
可能な限り、答えると仕様が前進する質問に寄せている。

## 2. コアルール関連

- 判定方式は最終的に `2D6` 系で行きたいですか。それとも `%` や d20 もまだ候補ですか。
  - 返答:
    - 確定: `2D6`
- 成功段階は `大成功 / 成功 / 失敗` の3段階で十分ですか。部分成功を独立させたいですか。
  - 返答:
    - 確定: `大成功 / 成功 / 部分成功 / 失敗` の4段階
- 戦闘は「依頼中の1障害」として軽く扱いたいですか。それとも独立した見せ場にしたいですか。
  - 返答:
    - 確定: 依頼中の1障害として軽く扱う
- セッション1回の長さは、短編依頼1本を想定していますか。それとも数件をまとめて回す想定ですか。
  - 返答:
    - 確定: 軽めの短編依頼1本
    - 確定: 想定時間は 15 分から 30 分程度

## 11. 次段階で確認したい質問

以下は、判定ルール詳細、キャラ生成、関係値、負傷・ストレス運用の叩き台を固めるための質問である。

### 11.1 判定ルール詳細

- 標準目標値は `6-10` の 5 段階でよさそうですか。それとももっと狭くしたいですか。
- 補正幅は原則 `-2` から `+2` 程度で十分そうですか。
- 部分成功の代償は、`時間 / 物資 / 状態悪化 / 関係悪化 / 情報不足 / 成果不完全` あたりを基本にしてよさそうですか。
- 大成功の利益は、`報酬増加 / 関係改善 / 追加情報 / 状態回復 / 次判定有利` あたりで足りますか。
- ゾロ目や出目そのものに特別ルールを入れたい気持ちはありますか。それとも今は入れない方がよいですか。

### 11.2 キャラ生成

- 能力値の初期配分は、`2,2,1,1,0` や `3,1,1,1,0` のような型から選ぶ形でよさそうですか。
- 能力値の初期合計点は `6` と `7` のどちらがしっくり来ますか。
- 初期特技数は `2` と `3` のどちらがよさそうですか。
- 初期欠点数は、標準 `1`、重めで `2` くらいでよさそうですか。
- 初期ギルド人数は、試運転段階では `4-5人` を基本にしてよさそうですか。

### 11.3 関係値運用

- 関係値は `-2 から +2` の 5 段階とタグ併用でよさそうですか。それとももっと軽くしたいですか。
- 初期関係は「良好が少し、中立が多く、悪関係は少数」くらいでよさそうですか。
- 関係が悪い相手を同じ依頼に出したとき、明確に不利補正が付く形でよいですか。
- 関係変化は 1 依頼につき原則 1 段階変動くらいで抑える方向でよいですか。
- ギルド外の NPC や勢力との関係も、キャラ個人に紐づけて持たせたいですか。

### 11.4 負傷・ストレス・休養

- 負傷とストレスは、どちらも `0-3` の 4 段階管理でよさそうですか。
- 負傷とストレスは別管理でよさそうですか。それともどちらかを簡略化したいですか。
- 自然回復はどの程度入れたいですか。`1 セッション休めば 1 段階回復` くらいでよさそうですか。
- 休養中の個別イベントや会話でストレス回復が起きる構造は入れたいですか。
- 重度状態のキャラは「参加できない」まで行かせたいですか。それとも強い不利に留めたいですか。
