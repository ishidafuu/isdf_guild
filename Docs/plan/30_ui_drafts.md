## 17. 画面草案（1つ目: 日中運用の3カラム）

### レイアウト

- 左カラム: 依頼ボード（新着/保留/撤回、公開範囲、信頼度メモ）
- 中央カラム: 冒険者一覧（待機中/出発中/離脱中、個別面談の入口）
- 右カラム: 出発待ち/進行中案件/帰還予定（残り日数、要注意フラグ）

### 基本操作

- 依頼をクリック: 詳細確認 -> 掲示方針を変更（全体/限定/保留）
- 冒険者をクリック: 個別面談を開く -> 希望/質問確認 -> 提案/割当
- 進行中案件をクリック: 現在状況と帰還見込みを確認
- 画面右上 `次の日`: 日付を1日進め、帰還報告を発生
- 冒険者の応募通知: 応募理由の要約を確認し、面談へ移動

### 作業感を抑えるための表示ルール

- 常に「今判断が必要なもの」を上位表示（新着依頼、帰還報告、未対応質問）
- 件数が多いときはタブで圧縮（依頼: 新着/進行/保留、冒険者: 待機/出発中/離脱中）
- 右カラムもタブ圧縮（出発待ち / 進行中 / 帰還報告）
- 日報はポップアップで要約3行のみ先表示し、詳細は任意展開

## 18. 画面草案（2つ目: 冒険者の個別面談）

### 面談画面で表示する情報

- 冒険者プロフィール: 経歴、資格、最近の実績、現在の気分/疲労
- 調査書要約: 生き方、禁忌、金銭観、口調などの短い人物メモ
- 人物年表の抜粋: 直近5件の転機（成功/負傷/不信/見舞い/相棒化など）
- 冒険者の自己申告: 「この依頼は得意/不安」などの主観コメント
- ギルド側メモ: 過去配属との相性、注意点、依頼主との関係
- 候補依頼: この冒険者に提案可能な依頼一覧（公開範囲内）
- 関係バッジ: 同行候補との `相棒候補 / 健全ライバル / 険悪ライバル / 不信` を簡易表示

### 面談中に選べる行動

- 質問に答える: 依頼内容の追加説明、報酬/危険説明
- 提案する: 特定依頼への応募を打診する
- 見送る: 今回は配属せず待機を選ぶ
- 助言する: 装備方針、同行推奨、撤退優先などの出発前方針
- 説得する: おだてる / 応援する / たきつける などの働きかけ
- 打ち切られる: 冒険者が「もういい」を選ぶと面談終了

### 面談結果として保存する内容

- 冒険者の意思: 受諾/保留/辞退
- 信頼変化メモ: 納得感が高い説明だったかどうか
- 次回面談フラグ: 再提案したい依頼、要注意条件
- 発言要約: 面談内容の要点だけを短文保存
- 打ち切り理由要約: 冒険者側が面談を終えた理由（短文）

### UI方針（作業感対策）

- 1画面1人に集中し、同時に大量情報を見せない
- 重要情報（受諾可否に影響する要素）を上段固定表示
- 面談ログは短文で蓄積し、台帳から後で参照可能にする

## 19. 画面草案（3つ目: 帰還時の日報テンプレ）

### 1. 要約3行の固定フォーマット

- 1行目（結論）: `依頼名 / 成否 / 主要成果`
- 2行目（被害）: `パーティ状態（無事・負傷・長期離脱）/ 消耗状況`
- 3行目（余波）: `依頼主反応 / 評判変化の要点 / 次回メモ`

### 2. 詳細ログに必ず載せる項目

- 依頼情報: 依頼名、依頼主、受注日、帰還日
- 配属情報: 参加冒険者、役割、出発前助言
- 結果情報: 成功/部分成功/失敗/惨敗、獲得物、損耗、想定外イベント
- 反応情報: 依頼主コメント、冒険者コメント
- 更新情報: 評判変化、個人履歴への追記内容

### 3. 要注意アラートの表示条件

- 冒険者が長期離脱または退団した
- 依頼主側または冒険者側の評判が大きく低下した
- 同種依頼で失敗が連続した
- 申告情報と実態の乖離が大きい依頼主が判明した

### 4. 評判・個人履歴への反映ルール（文言テンプレ）

- 評判（依頼主側）: `適切性 +/-, 倫理性 +/-, 収益性 +/-`
- 評判（冒険者側）: `安全配慮 +/-, 公正配分 +/-, 成長機会 +/-`
- 個人履歴（冒険者）: `誰の依頼で / 何を担当し / どう感じたか` を短文で追記
- メモ方針: 失敗時も責任追及のみで終わらせず、次回判断に使える学習文を残す

## 20. 画面草案（4つ目: 冒険者台帳 / 人物年表）

### レイアウト

- 左: 現在状態（待機/出発中/離脱中、気分、疲労、最近の悩み）
- 中央: 時系列の人物年表（1行見出し + 展開で詳細）
- 右: 関係バッジ、実務評価ラベル、最近変わった傾向

### 年表に載せるイベント

- ミッション結果: 成否、担当、被害、本人の受け止め
- 負傷/長期離脱: 離脱理由、復帰予定、ギルドの見舞い有無
- 面談要約: 何に不満/納得したか、何を保留したか
- 関係変化: 相棒候補化、ライバル化、不信の発生
- 依頼主反応: 褒められた、警戒された、再指名された など

### UI方針

- 数値の生表示は最小限にし、`最近かなり慎重`, `一仕事越えて自信がついた` のような文章ラベルを優先する
- 低重要度イベントは折りたたみ、転機イベントだけを常に上位表示する
- 年表から当時の日報/面談ログ/関係イベントへ飛べるようにする
- `volatile_hook` は年表上の最新未消化イベントから自動で引く
