# 世界観レビュー記録

最終更新: 2026-03-08

## 0. この章の目的

- 世界観構築時にどの観点で何を確認したかを残す
- 破綻しやすい論点を先に可視化し、採否を記録する
- 今後の世界観追加時のレビュー観点を使い回せるようにする

## 1. レビュー体制

今回は次の3視点で世界観をレビューした。

### レビューA: システム整合担当

- 重点: ゲームループ、依頼供給、評判、ギルド制度との噛み合い

### レビューB: 物語整合担当

- 重点: 歴史、勢力、地理、文化の因果と世界らしさ

### レビューC: 生成運用担当

- 重点: 自動生成NPC、依頼テンプレート、将来のイベント拡張耐性

## 2. レビューA: システム整合担当

### 指摘1: 新設ギルドが最初の依頼を得る導線が弱い

- 問題:
- 新設参入が制度上可能でも、実績ゼロのギルドに依頼が来る理由が薄いと序盤が不自然になる

- 対応:
- `05_world_setting.md` に、既存ギルドの偏りと「拾われない案件」があることをより明確に残す
- `06_regional_structure.md` に、地方帯ごとの小口・中口案件が新設ギルドへ流れうる構造を入れる

- 採否:
- 採用

### 指摘2: 地域と依頼カテゴリの結びつきはできているが、季節変動がない

- 問題:
- 長期運営型である以上、常に同じ頻度で案件が出ると世界が平板になる

- 対応:
- `06_regional_structure.md` に季節要因を追加し、収穫期、巡礼期、湿地採取期で案件傾向が変わる前提を足す

- 採否:
- 採用

### 指摘3: 上位権力の影が弱い

- 問題:
- 都市国家的とだけ書くと、なぜより大きな権力が全部統制しないのかが曖昧

- 対応:
- エンベルクが `盟約都市` であり、広域の宗主権はあるが実務自治が強いことを追加する

- 採否:
- 採用

## 3. レビューB: 物語整合担当

### 指摘1: 遺跡と巡礼の二本柱は良いが、両者の歴史的関係が弱い

- 問題:
- 旧塔時代の遺構と、現在の聖火教会の導線が無関係すぎると世界が分断して見える

- 対応:
- 教会は旧塔時代の遺構を完全継承していないが、一部を聖地化して再解釈したという整理に寄せる

- 採否:
- 採用

### 指摘2: エンベルク圏の文化差が少ない

- 問題:
- 地方帯ごとの産業差はあるが、人々の考え方の違いが弱いと交渉依頼が薄くなる

- 対応:
- 各地方帯に「気風」を追加し、村、河運、巡礼宿、遺跡周辺の人間の判断基準を分ける

- 採否:
- 採用

### 指摘3: モンスターが単なる障害物になりやすい

- 問題:
- 地域の文脈と結びつかないと、結局どこでも同じ討伐になる

- 対応:
- 地域ごとに「なぜその危険がその土地で発生しやすいか」を補足し、遺跡、湿地、街道、巡礼路と結びつける

- 採否:
- 採用

## 4. レビューC: 生成運用担当

### 指摘1: 自動生成NPCの所属先と口実がまだ弱い

- 問題:
- 人物生成時に「どの地域の、どの立場で、何に困っているか」が弱いと量産文が平板になる

- 対応:
- 依頼主生成の基礎単位として `所属地域 / 所属組織 / 当面の困りごと` を固定する前提を明文化する

- 採否:
- 採用

### 指摘2: 依頼テンプレの再利用単位が未定義

- 問題:
- 毎回完全新規に依頼を作ると、世界観の一貫性を保ちにくい

- 対応:
- 今後は `地域帯 × 依頼カテゴリ × 典型的揉め方` をテンプレート最小単位にする

- 採否:
- 採用

### 指摘3: 競合ギルドの色がまだ抽象的

- 問題:
- 他ギルド達成や案件流出を生成的に扱うには、競合ギルドの方針タグが必要

- 対応:
- これは次段階で `他ギルド類型テンプレート` として切り出す

- 採否:
- 保留。次章で対応する

## 5. 今回のレビューで固定した方針

- エンベルクは完全独立国家ではなく、広域秩序の中で自治を保つ盟約都市とする
- 旧塔時代の遺構と聖火教会の聖地運用は、断絶ではなく再解釈関係にある
- 地方帯には産業差だけでなく気風差を持たせる
- 依頼テンプレートは `地域帯 × 依頼カテゴリ × 典型揉め方` を骨格にする
- 季節要因を今後の依頼変動設計へ接続する

## 6. 残課題

- 他ギルドの類型テンプレート
- 依頼主カテゴリテンプレート
- 冒険者出自テンプレート
- 季節変動をどの程度MVPへ入れるか
